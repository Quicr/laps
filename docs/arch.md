# Latency Aware Publish/Subscribe (LAPS) Architecture

## Media Over QUIC Transport
LAPS implements [MoQT](https://datatracker.ietf.org/doc/html/draft-ietf-moq-transport-07) client side connections to support both publishers and subscribers.

LAPS is a relay implementation that supports MoQT clients and interconnectivity utilizing a proprietary peering
protocol.
